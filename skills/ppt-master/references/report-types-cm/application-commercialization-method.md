# China Mobile Application Commercialization Method

Use this file for China Mobile-style `technical_research` Chapter 6: 应用与落地.

This file does not invent facts. It provides the method lenses that should be applied to source material.

## Method Lenses

Analyze the chapter through these lenses:

1. scenario or case entry point
2. pain point or demand trigger
3. technical capability mapping
4. value translation
5. landing path or implementation route
6. constraints, dependencies, and maturity

## What To Extract

- concrete scenarios or cases
- the real user or business problem
- which technical capability addresses that problem
- what value is created
- what must be in place to land it
- what evidence supports the judgment

## Required Method-to-Source Mapping

Before outlining the chapter, produce a mapping that states:

- which lenses are used
- which source facts support each lens
- which lenses lack evidence and must be marked `[待补充]`

## Organizing Choices

Choose the chapter logic based on source richness:

- multiple cases
- one or two core scenarios
- value chain
- landing path
- risk and maturity

Do not force a fixed checklist if the source only supports a narrower reading.

## Output Bias

- prefer scenario-level specificity
- prefer value translation over broad industry labels
- keep final judgments visible
- mark unsupported claims as `[待补充]`
