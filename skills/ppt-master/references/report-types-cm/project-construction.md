# China Mobile Report Type: `project_construction`

Use this file when the deck is a China Mobile-style construction plan, feasibility deck, implementation proposal, or approval briefing.

## Default Chapter Framework

1. 项目背景与必要性
2. 建设目标与范围
3. 总体方案与架构
4. 实施路径与阶段计划
5. 投资估算与资源安排
6. 风险问题与保障机制
7. 决策事项与建议

## What Each Chapter Must Answer

### 1. 项目背景与必要性

- why the project exists
- what pain points or policy drivers support it
- why now

### 2. 建设目标与范围

- what the project is trying to achieve
- what is in scope and out of scope

### 3. 总体方案与架构

- how the scheme is structured
- what major modules or capabilities are included

### 4. 实施路径与阶段计划

- what the phased roadmap looks like
- what happens in each stage

### 5. 投资估算与资源安排

- what resources are required
- what the budget logic is

### 6. 风险问题与保障机制

- what can go wrong
- what governance or support mechanisms exist

### 7. 决策事项与建议

- what needs approval
- what action is recommended

## Extraction Priorities

- necessity statements
- objectives and scope splits
- architecture blocks
- timeline and stage plans
- budget and resource numbers
- decision items
- risks and safeguards

## Recommended Patterns

- metrics cluster + support modules
- architecture row + explanation blocks
- timeline + status blocks
- comparison table + verdict sidebar
- summary cards + action grid

## Page-Level Judgment Language

Prefer:

- necessity judgment
- scope decision
- implementation priority
- resource implication
- risk control judgment
- approval recommendation
