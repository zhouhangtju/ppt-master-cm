# China Mobile Design Language

This file defines the shared China Mobile reporting language for PPT Master.

Use it when the user explicitly asks for China Mobile style, internal operator briefing style, or when the deck clearly targets China Mobile-style internal reporting.

## Positioning

- internal enterprise reporting, not public marketing
- blue-white base with restrained red emphasis
- high information density with clear structure
- conclusion-first, judgment-visible, modular composition
- more evidence and containers, less decorative whitespace

## Style Priority

Resolve style in this order:

1. user-provided China Mobile reference deck or screenshots
2. deck-specific brand or department requirements
3. this shared China Mobile baseline

Do not flatten a mature reference deck into a weaker generic blue-white slide.

## Color System

| Role | Value | Use |
| --- | --- | --- |
| Primary blue | `#0050B3` | section blocks, page anchors, strong labels |
| Bright blue | `#00B4D8` | guide lines, active emphasis, highlights |
| Deep blue | `#003366` | dark backgrounds, cover/chapter pages |
| Soft blue | `#E6F4FF` | background panels, light cards |
| Judgment red | `#D93A49` | critical numbers, risks, final judgments |
| Primary text | `#1A1A1A` | titles and body |
| Secondary text | `#4A5568` | support text |
| Divider | `#D9E2EC` | separators and grid lines |

Rules:

- blue defines structure
- red is reserved for decisive emphasis, not for decoration
- keep the deck within a restrained enterprise palette

## Typography

- Chinese default: `Microsoft YaHei`
- English / numbers default: `Arial`
- body text should stay compact and readable
- titles should be strong but not oversized

Typical bias:

- page title: `28-34px`
- module title: `20-24px`
- body: `16-18px`
- annotation: `12-14px`

## Layout Grammar

- one page should carry one core judgment
- most pages should have a visible judgment strip, takeaway bar, or observation sentence
- chapter pages should act as directory-style transitions
- content pages should prefer evidence modules, comparison frames, process blocks, case grids, and KPI clusters
- avoid pages that are only several equal-weight empty white cards

## Density Bias

China Mobile style is dense-first:

- prefer structure over empty space
- prefer adding evidence modules over inflating typography
- prefer splitting the page into meaningful modules over dumping long paragraphs
- when content is rich, use 2-column or 3-column evidence layouts aggressively

## Directory Pages

Each chapter should have a directory-style chapter page.

Required traits:

- visible chapter list or structural positioning
- strong current-chapter emphasis
- weakened non-current chapters
- clear left-side or center structural device

## Content Writing Bias

- conclusion first
- evidence supports judgment
- numbers stay visible
- comparisons must have explicit verdicts
- risks and blockers should be named directly
