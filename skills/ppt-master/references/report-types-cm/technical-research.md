# China Mobile Report Type: `technical_research`

Use this file when the deck is a China Mobile-style technical research, capability study, architecture analysis, model evaluation, or landing judgment report.

## Default Chapter Framework

1. 概述与简介
2. 背景与发展
3. 原理与架构
4. 能力特点分析
5. 竞争力分析
6. 应用与落地
7. 总结与建议

## What Each Chapter Must Answer

### 1. 概述与简介

- what is being researched
- why it matters now
- what the report will judge

### 2. 背景与发展

- what technical or industry context led here
- what trend or pressure drives current attention
- why this topic deserves focus now

### 3. 原理与架构

- how the mechanism works
- what the core architecture is
- how the technical chain closes

### 4. 能力特点分析

- what the subject does well
- what changed versus previous approaches
- what tasks or scenarios fit best

### 5. 竞争力分析

- how it compares with alternatives
- where it is stronger or weaker
- what constraints or risks matter

### 6. 应用与落地

- how capability becomes business or customer value
- what the main landing paths are
- what supports or blocks implementation

### 7. 总结与建议

- what the overall judgment is
- what to track, adopt, avoid, or prioritize

## Extraction Priorities

- technical concepts and mechanisms
- architecture layers
- capability changes
- differentiated comparisons
- application cases
- landing paths
- risk and constraints
- final judgments

## Recommended Patterns

- research evidence mosaic
- architecture row + explanation blocks
- comparison table + verdict sidebar
- observation bar + case grid + judgment strip
- metrics cluster + support modules
- summary cards + action grid

## Page-Level Judgment Language

Every page should close with one of:

- core conclusion
- major difference
- key value
- best-fit scenario
- main landing barrier
- maturity judgment

## Chapter 6 Rule: 应用与落地

When Chapter 6 exists:

- combine source material with `application-commercialization-method.md`
- do not generate the chapter from source material alone
- produce an explicit method-to-source mapping first
- mark unsupported dimensions as `[待补充]`
- prefer case, scenario, value-chain, landing-path, or risk logic over broad industry labels
