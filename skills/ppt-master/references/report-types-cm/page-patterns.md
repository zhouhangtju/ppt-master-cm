# China Mobile Page Patterns

Use these approved page patterns when a deck follows the China Mobile reporting style.

## Pattern Selection Order

1. identify the page's core judgment
2. identify the dominant information type
3. check whether a reference page should be inherited
4. choose one approved pattern below

## 1. Directory Chapter Page

Use for:

- every chapter transition

Required:

- full chapter list or equivalent chapter positioning
- current chapter strongly highlighted
- non-current chapters weakened

Avoid:

- plain text chapter list without hierarchy

## 2. Research Evidence Mosaic

Use for:

- trend, background, policy, progress, evidence-heavy pages

Structure:

- conclusion title
- 3-6 evidence blocks
- local title per evidence block
- page-level takeaway

Avoid:

- turning rich evidence into long paragraphs

## 3. Observation Bar + Case Grid + Judgment Strip

Use for:

- application, landing, scenario, case pages

Structure:

- title + lead sentence
- top observation / watchpoint bar
- middle case cards or grouped modules
- bottom judgment strip

Avoid:

- several generic cards with no page-level judgment

## 4. Comparison Table + Verdict Sidebar

Use for:

- competitive analysis
- solution comparison
- scheme selection

Structure:

- comparison table or matrix
- side verdict or implication block
- strongest option visibly emphasized

## 5. Architecture Row + Explanation Blocks

Use for:

- mechanism explanation
- architecture evolution
- capability structure

Structure:

- top or center structural device
- 2-4 explanation modules
- one final technical judgment

## 6. Metrics Cluster + Support Modules

Use for:

- overview pages
- KPI summaries
- progress snapshots

Structure:

- 2-4 highlighted metrics
- support cards around them
- visible conclusion

## 7. Timeline + Status Blocks

Use for:

- project progress
- milestone review
- phased construction plans

Structure:

- milestone timeline
- status / owner / issue blocks
- next-step action area

## 8. Summary Cards + Action Grid

Use for:

- closing summary
- recommendation pages
- management decision pages

Structure:

- upper summary cards
- lower action / recommendation grid
- explicit distinction between current state and next move

## Weak Layouts To Avoid

- large title plus a few empty white boxes
- application pages with only broad industry labels
- comparison pages with no verdict area
- progress pages with no milestone or action structure
- sparse pages that shrink content strength instead of improving structure
