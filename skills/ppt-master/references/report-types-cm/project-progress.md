# China Mobile Report Type: `project_progress`

Use this file when the deck is a China Mobile-style milestone review, status update, coordination report, or progress briefing.

## Default Chapter Framework

1. 项目概况与目标回顾
2. 总体进展与阶段结论
3. 里程碑完成情况
4. 资源与能力准备情况
5. 问题风险与影响判断
6. 下一步计划与协同事项

## What Each Chapter Must Answer

### 1. 项目概况与目标回顾

- what the project is
- what target it is measured against

### 2. 总体进展与阶段结论

- where the project stands overall
- what the stage-level conclusion is

### 3. 里程碑完成情况

- what was completed
- what remains pending

### 4. 资源与能力准备情况

- what readiness exists across people, systems, data, or operations

### 5. 问题风险与影响判断

- what issues and risks matter
- what impact they have

### 6. 下一步计划与协同事项

- what happens next
- what coordination or decisions are needed

## Extraction Priorities

- milestone status
- completion percentages
- delays and blockers
- resource readiness
- risk statements
- action items and owners

## Recommended Patterns

- metrics cluster + support modules
- timeline + status blocks
- comparison table + verdict sidebar
- research evidence mosaic
- summary cards + action grid

## Page-Level Judgment Language

Prefer:

- current status judgment
- milestone conclusion
- readiness judgment
- blocker severity
- next-step action requirement
