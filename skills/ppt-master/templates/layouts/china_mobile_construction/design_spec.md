# China Mobile Project Construction Template - Design Specification

> Suitable for China Mobile project construction plans, feasibility presentations, investment request decks, and internal scheme briefings.

---

## I. Template Overview

| Property | Description |
| --- | --- |
| **Template Name** | china_mobile_construction |
| **Use Cases** | Project initiation, construction planning, investment estimation, approval briefings |
| **Design Tone** | Authoritative, structured, restrained, decision-oriented, enterprise-telecom |
| **Theme Mode** | Light theme (white background + China Mobile blue accents) |

---

## II. Canvas Specification

| Property | Value |
| --- | --- |
| **Format** | Standard 16:9 |
| **Dimensions** | 1280 x 720 px |
| **viewBox** | `0 0 1280 720` |
| **Page Margins** | Left/Right 60px, Top 80px, Bottom 40px |
| **Safe Area** | x: 60-1220, y: 80-680 |

---

## III. Design Positioning

- Use the same visual shell as `china_mobile_research` for first-version consistency
- Bias content layouts toward plan structure, scope split, timeline, investment, and decision items
- Favor orderly block modules and process structures over decorative compositions

---

## IV. Color and Typography

- Primary blue: `#0050B3`
- Support blue: `#00B4D8`
- Deep blue: `#003366`
- Text: `#1A1A1A`
- Font stack: `"Microsoft YaHei", "微软雅黑", SimHei, Arial, sans-serif`

---

## V. Page Types

- `01_cover.svg`
- `02_toc.svg`
- `02_chapter.svg`
- `03_content.svg`
- `04_ending.svg`

---

## VI. Construction Slide Expectations

- One page should answer one decision question
- Scope, necessity, goals, architecture, plan, budget, and decision items should each have visible module structure
- Avoid sparse pages with oversized titles
- Where source materials are dense, prefer process flows, split modules, and sidebars instead of shrinking type

---

## VII. Placeholder Specification

Use the same placeholder vocabulary as `china_mobile_research`.
