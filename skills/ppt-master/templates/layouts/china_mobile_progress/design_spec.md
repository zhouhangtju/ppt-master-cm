# China Mobile Project Progress Template - Design Specification

> Suitable for China Mobile project progress reports, milestone reviews, coordination briefings, and internal status updates.

---

## I. Template Overview

| Property | Description |
| --- | --- |
| **Template Name** | china_mobile_progress |
| **Use Cases** | Milestone review, progress status, issue tracking, next-step planning |
| **Design Tone** | Structured, concise, progress-oriented, information-dense, enterprise-telecom |
| **Theme Mode** | Light theme (white background + China Mobile blue accents) |

---

## II. Canvas Specification

| Property | Value |
| --- | --- |
| **Format** | Standard 16:9 |
| **Dimensions** | 1280 x 720 px |
| **viewBox** | `0 0 1280 720` |
| **Page Margins** | Left/Right 60px, Top 80px, Bottom 40px |
| **Safe Area** | x: 60-1220, y: 80-680 |

---

## III. Design Positioning

- Use the same visual shell as `china_mobile_research` for first-version consistency
- Bias content layouts toward milestone status, resource readiness, problem/risk, and next-step plan
- Encourage timeline, status-card, issue-grid, and action-list page structures

---

## IV. Color and Typography

- Primary blue: `#0050B3`
- Support blue: `#00B4D8`
- Deep blue: `#003366`
- Text: `#1A1A1A`
- Font stack: `"Microsoft YaHei", "微软雅黑", SimHei, Arial, sans-serif`

---

## V. Page Types

- `01_cover.svg`
- `02_toc.svg`
- `02_chapter.svg`
- `03_content.svg`
- `04_ending.svg`

---

## VI. Progress Slide Expectations

- Every progress page should end with a visible status judgment or action takeaway
- Prefer compact status modules, milestone lines, issue-risk tables, and next-step cards
- Avoid turning progress content into long prose paragraphs
- When information is thin, add structure; do not pad with whitespace

---

## VII. Placeholder Specification

Use the same placeholder vocabulary as `china_mobile_research`.
